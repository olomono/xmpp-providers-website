Providers

Categories describe how well a provider is suited for registration and usage. Providers of categories A and B provide a good chat experience, while C provides an average, and D the least good chat experience.

Category  A

Providers in this category can be used for a registration without user interaction. Users can simply confirm to register on a randomly chosen provider and start to chat right away. Those providers must have the best properties and support registrations via XMPP apps.

{{ partial "provider-list-items.html" (where .Site.Data.providers ".category" "==" "A") }}

Category  B

Providers in this category can be used for registration with user interaction. Users can select a provider based on individual preferences (e.g., the provider's name). Those providers must have the best properties but do only need to support registrations via a web page.

{{ partial "provider-list-items.html" (where .Site.Data.providers ".category" "==" "B") }}

Category  C

Providers in this category can be used for web registrations but do only need to have average properties. Those providers should only be offered to users if there is a certain reason for it.

{{ partial "provider-list-items.html" (where .Site.Data.providers ".category" "==" "C") }}

Category  D

Providers in this category cannot, if no registration is supported, or should not, because of bad properties, be used for registrations. Those providers should only be used for autocomplete (e.g., while adding a contact).
